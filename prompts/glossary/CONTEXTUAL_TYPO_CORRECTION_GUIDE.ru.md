# Руководство по контекстной правке опечаток и STT-ошибок (для нейросети)

> **ОБЯЗАТЕЛЬНО К ПРОЧТЕНИЮ** перед любой правкой расшифровок речи, OCR и черновиков.  
> Подключается вместе с [`../ru/UNIVERSAL_EDITOR_SYSTEM.md`](../ru/UNIVERSAL_EDITOR_SYSTEM.md) и профилем [`../ru/profiles/SPEECH_TO_TEXT.md`](../ru/profiles/SPEECH_TO_TEXT.md).

Другие языки: [EN](CONTEXTUAL_TYPO_CORRECTION_GUIDE.en.md) · [UK](CONTEXTUAL_TYPO_CORRECTION_GUIDE.uk.md)

---

## Железное правило (не нарушать)

> **Исправляй слова и фразы ТОЛЬКО в контексте — никогда по отдельности, как автокорректор.**

| Запрещено | Почему |
|-----------|--------|
| Пробегать текст и «чинить» каждое подозрительное слово по словарю | Омофоны и STT-ошибки часто **грамматически корректны**, но **семантически неверны** |
| Делать глобальную замену `X → Y` по всему файлу | Одно и то же «услышанное» слово в разных местах означает разное |
| Доверять спеллчекеру / SymSpell / Hunspell без контекста | Они ранжируют по орфографии, а не по **смыслу реплики** |
| «Угадывать» смысл молча, если контекст не сходится | Лучше оставить исходник и спросить пользователя |

**Минимальный контекст для решения:**
1. **целое предложение** (лучше — абзац или пара соседних реплик);
2. **тема фрагмента** (быт, здоровье, IT, отношения, работа…);
3. **кто говорит** (имя спикера, если есть);
4. **глоссарий проекта** (`config/glossary_user.json`).

Если после анализа контекста остаётся **2+ правдоподобных чтения** — **не правь**; вынеси в отчёт или задай вопрос.

## Как правильно (алгоритм для ИИ)

```
1. Прочитай абзац / реплику целиком — пойми намерение говорящего.
2. Отметь «аномалии»: бессмыслица, ломаная коллокация, сбой согласования, чужой жаргон.
3. Сформулируй 2–5 кандидатов — не одно «слово из таблицы», а целую фразу.
4. Выбери вариант, при котором:
   - предложение звучит естественно устно;
   - согласуются падежи, время, лицо;
   - не ломается соседняя реплика (вопрос ↔ ответ).
5. Если уверенность < 80% — оставь исходник + пометка [?] или вопрос пользователю.
```

**Пример (контекст решает):**

| Исходник STT | Плохо (без контекста) | Хорошо (в контексте) |
|--------------|----------------------|----------------------|
| «расставлю температура короче валясь» | «рассматриваю в общем валяюсь» | «У меня температура, я валяюсь» (+ отдельно про разъём, если речь о технике) |
| «надо купить мышь» | «мышь» → «мышца» | Оставить «мышь», если речь о компьютере; «мышь-животное» — если про зоомагазин |
| «залил в прод» | «залил в продовольственный» | «задеплоил в production» — если IT-диалог |

## Типы ошибок (что искать)

### 1. Омофоны и слуховые подстановки (STT / диктовка)
Звучат похоже, пишутся по-разному: *мир / мир*, *написал / напечатал*, *коса / коса*.

### 2. Слитно / раздельно из-за пауз
*«не знаю что делать»*, *«вообщем»*, *«ни чего»* — иногда это стиль речи, иногда ошибка; смотри на смысл.

### 3. Имена, бренды, топонимы
Whisper часто ломает редкие имена: *Амфи → Анфи*, *Кир → Kir* — **только по глоссарию пользователя**, не из чужих проектов.

### 4. Code-switching и IT-жаргон
Русская речь + английские термины; модель может выдать кириллическую «фонетику» или обычное слово вместо термина (*кафка → Kafka*, *гошечка → Go*).

### 5. Галлюцинации Whisper
Вставки, которых не было: «подписывайтесь на канал», «продолжение следует», субтитры, мусор на другом языке — **удалять без вопросов**.

### 6. OCR / опечатки клавиатуры
Перестановки соседних букв (*теиз → это*), *е/ё*, *и/й*, *ш/щ* — править, если слово **не существует в данном контексте**.

### 7. Омографы (одинаковое написание — разный смысл)
Ударение не видно в тексте: *замок / замок*, *мука / мука*, *белки / белки*. Решение **только по соседним словам**.

## Открытые ресурсы (для понимания, не для слепой подстановки)

Используй как **справочник кандидатов**, не как скрипт замены.

| Ресурс | Что даёт | Ссылка |
|--------|----------|--------|
| **Kartaslov orfo_and_typos** | Частотные орфо- и фонетические опечатки RU | https://github.com/dkulagin/kartaslov/tree/master/dataset/orfo_and_typos |
| **Silero homographs** | Список русских омографов (разное ударение → разный смысл) | https://github.com/snakers4/silero-stress/wiki/Homograph-List |
| **more_russian_homophones** | Пары омофонов (датасет) | https://huggingface.co/datasets/kkitt/more_russian_homophones |
| **typos-misspellings-homophones** | Опечатки + омофоны (EN-ориентир, полезен для mixed text) | https://huggingface.co/datasets/guychuk/typos-misspellings-homophones-dataset |
| **ai-forever/spellcheck_benchmark** | Пары «ошибка → исправление» в контексте предложений RU | https://huggingface.co/datasets/ai-forever/spellcheck_benchmark |
| **JamSpell** | Контекстный спеллчекер (n-gram); `FixFragment`, не по одному слову | https://github.com/bakwc/JamSpell |
| **Yandex.Speller** | API орфографии; всё равно проверяй смысл вручную | https://yandex.ru/dev/speller/ |
| **DeepPavlov spelling** | Noisy-channel + языковая модель | https://docs.deeppavlov.ai/en/master/features/models/spelling_correction.html |
| **SpellRuEval** | Бенчмарк русской коррекции | http://www.dialog-21.ru/en/evaluation/2016/spelling_correction/ |

**Вывод для ИИ:** даже JamSpell и Yandex.Speller ошибаются на омофонах и жаргоне. Твоя задача — **редактор с пониманием смысла**, а не пайплайн `replace()`.

## Частые STT-пары (кандидаты — не автозамена!)

> В колонке «Когда верно» указан **контекст**. Без него — не меняй.

### Быт, здоровье, эмоции

| Услышано (примеры) | Возможно имелось в виду | Когда верно |
|--------------------|-------------------------|-------------|
| валясь / валяюсь | валяюсь (лежу) | болезнь, усталость, дом |
| температура / температуру | температура (тело) | плохое самочувствие, не погода |
| лекарство / лекарства | то же | аптека, врач |
| разъём / разьём / разъем | разъём | техника рядом: зарядка, кабель |
| голова болит / болеет голова | то же | перефраз, не «голова болеет» в смысле болезни головы как органа |
| давление скачет | гипертония / стресс | здоровье |
| короче | «короче говоря» (разговор.) | сохранять как маркер устности, не «короче = ножницы» |

### IT, техника, бренды

| Услышано | Возможно | Контекст |
|----------|----------|----------|
| юсб / те из бы / ю эс би | USB | кабель, флешка, порт |
| ссд / эс эс д | SSD | диск, накопитель |
| винда / виндовс | Windows | ОС, компьютер |
| блютуз / блю ту з | Bluetooth | наушники, колонка |
| вай фай / вайфай | Wi-Fi | интернет, роутер |
| телега / телеграм | Telegram | мессенджер, чат |
| эпл / эппл | Apple | iPhone, Mac |
| кафка / кафку | Kafka | бэкенд, очереди, IT |
| докер / докер контейнер | Docker | деплой, DevOps |
| гит / гитхаб | Git / GitHub | код, репозиторий |
| прод / на прод | production | деплой, релиз |
| дев / девелоп | dev / development | среда разработки |
| линукс / линуксе | Linux | сервер, ОС |
| питон / пайтон | Python | язык программирования |
| джава / жаба | Java | язык (жаргон — оставить, если стиль диалога) |
| реакт / реакт js | React | фронтенд |
| кубер / кубернетес | Kubernetes | оркестрация |
| нжинкс / энджинкс | Nginx | веб-сервер |
| постгрес / постгре | PostgreSQL | БД |
| редис | Redis | кэш |
| эс кью эл / скул | SQL | запросы, БД |

### Разговорные и «короче»

| Услышано | Возможно | Примечание |
|----------|----------|------------|
| ну типа / типа того | разговорная связка | не вычищать без просьбы |
| как бы | паразит | убрать умеренно, не там где несёт смысл |
| в общем | вводное | не путать с «валяюсь» и не заменять механически |
| щас / сейчас | сейчас | норма для живой речи |
| чё / что | что | стиль диалога |
| норм / нормально | нормально | |

### Омофоны (классика — только по смыслу фразы)

| Пара | Различие по контексту |
|------|------------------------|
| мир / мир | мир (покой) vs мир (планета) — смотри соседние слова |
| ключ / ключ | от двери vs музыкальный |
| лук / лук | овощ vs оружие |
| коса / коса | причёска vs инструмент |
| плата / плата | деньги vs печатная плата (IT) |
| файл / фал | файл vs опечатка «фал» |
| орёт / о рёте | кричит vs «о рёте реки» |
| написал / напечатал | синонимы в речи — не менять без причины |
| что бы / чтобы | союз цели vs «что бы ты ни сказал» |
| тоже / тож | тоже (правильно) vs опечатка |
| не / ни | частица отрицания vs «ни разу» |
| в след / вслед | «вслед за» vs ошибочное разделение |

## Матрица «похожих букв» (OCR / быстрый набор)

По данным Kartaslov и типичным опечаткам: часто путают **а↔о**, **е↔и**, **ь↔ъ**, **ш↔щ**, **п↔р**, **т↔ь**.  
Проверяй, существует ли слово **в этой теме**; если нет — ищи ближайшее по смыслу **во всём предложении**.

## Антипаттерны (реальные ошибки редактора-ИИ)

1. **`расставлю` → `рассматриваю`** только потому, что «так чаще встречается» — неверно без контекста планов/болезни.
2. **`валясь` → `в общем`** — разные слова; «валяюсь» часто верно при температуре.
3. **`прод` → `продовольственный`** в IT-чате — ломает смысл.
4. **`мышь` → `мышь` (животное)** в тексте про компьютер.
5. **Убрать мат / сленг** без запроса — нарушение контракта Vox2Book.
6. **Подставить имена из примеров глоссария** в чужой проект.
